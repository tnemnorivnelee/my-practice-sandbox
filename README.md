## 🔔 푸시 알림 테스트 가이드 (필독!)

Expo의 보안 정책 변경으로 인해, 스토어에서 다운로드한 **기본 'Expo Go' 앱에서는 더 이상 푸시 알림이 작동하지 않습니다.**
따라서 우리는 **'Development Build(개발용 빌드)'**라는 우리만의 전용 앱을 사용하여 개발합니다.

### 1. 준비물
* **실제 핸드폰** (에뮬레이터에서는 푸시 알림이 오지 않습니다!)
* **Android 폰 권장** (아이폰은 유료 개발자 계정이 등록된 기기만 가능합니다)
* PC와 핸드폰이 **동일한 Wi-Fi**에 연결되어 있어야 합니다.

### 2. '개발용 앱' 설치하기 (최초 1회)
기존의 Expo Go 앱이 아닌, **'my-practice-sandbox' (또는 hanse-ro)** 라는 이름의 전용 앱을 설치해야 합니다.

1.  [이 링크(EAS 빌드 페이지 링크를 여기에 넣으세요)]에서 최신 **Android Build (APK)**를 다운로드합니다.
2.  핸드폰으로 파일을 옮겨 **설치**합니다. ("출처를 알 수 없는 앱 허용" 필요)
3.  설치된 **'my-practice-sandbox'** 앱을 한번 켜서 잘 열리는지 확인합니다.

### 3. 개발 서버 실행하기 (명령어가 다릅니다!)
이제부터는 `yarn start` 대신 아래 명령어를 사용해야 이 전용 앱과 연결됩니다.

```bash
npx expo start --dev-client
```

1.  위 명령어나 `npx expo start --dev-client`를 터미널에 입력해 서버를 켭니다.
2.  핸드폰에서 방금 설치한 **'my-practice-sandbox' (또는 hanse-ro)** 앱을 실행합니다.
3.  터미널에 나타난 QR 코드를 스캔하거나, PC와 폰이 같은 Wi-Fi에 있다면 리스트에서 서버를 선택해 연결합니다.

---

### 4. 🔔 알림 발송 테스트 해보기
백엔드 없이도 푸시 알림이 잘 오는지 테스트할 수 있습니다.

1.  앱 하단의 **[알림 테스트]** (또는 연습) 탭으로 이동합니다.
2.  화면에 파란 글씨로 표시된 **`ExponentPushToken[...]`** 전체를 복사합니다.
3.  PC 브라우저에서 **[Expo 알림 발송 도구](https://expo.dev/notifications)** 사이트에 접속합니다.
4.  아래 내용을 각 항목에 입력하고 맨 아래 **Send a notification** 버튼을 누릅니다.

    * **To:** `(방금 복사한 토큰 붙여넣기)`
    * **Title:** `테스트 알림`
    * **Body:** `안녕하세요! 한세로 알림입니다.`
    * **Data (JSON):**
        ```json
        { "type": "CHAT", "roomId": "123" }
        ```

5.  핸드폰에 알림이 **"띠링!"** 하고 오면 성공입니다! 🎉
    * *(앱이 켜져 있으면 상단 배너로, 꺼져 있으면 시스템 알림으로 옵니다)*
